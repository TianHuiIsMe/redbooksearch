## 一、项目概述

本项目实现了一套基于AI Agent架构的小红书内容自主调研系统，能够模拟真实用户行为，自动化完成内容采集、智能分析、报告生成全流程。

### 核心特性

- ✅ **AI Agent智能体**: 实现「感知-思考-决策-执行」完整闭环
- ✅ **任务自主拆解**: 自然语言需求 → 自动拆解为多步骤子任务
- ✅ **真实用户模拟**: 基于Playwright，随机延迟、鼠标移动、滚动行为
- ✅ **AI智能分析**: 内容分类、广告甄别、情感分析、关键词提取
- ✅ **自动报告生成**: 结构化数据 + 统计维度 + 可视化结论
- ✅ **去重与增量**: 支持断点续传和增量采集
- ✅ **完整日志**: 记录Agent每一步思考和决策过程

## 三、安装与运行

### 3.1 快速安装

```bash
# 1. 进入项目目录
cd xiaohongshu_ai_agent

# 2. 安装依赖
pip install -r requirements.txt

# 3. 安装Playwright浏览器
playwright install chromium

# 4. 运行测试脚本验证安装
python test_installation.py
```

### 3.2 配置API密钥

编辑 `config/config.json`，填写您的OpenAI API密钥：

```json
{
  "ai": {
    "api_key": "YOUR_API_KEY_HERE",  // 替换为您的API密钥
    "model_provider": "openai",
    "model_name": "gpt-3.5-turbo"
  }
}
```

**注意**: 如果没有API密钥，可以使用演示模式（见3.3节）

### 3.3 运行演示模式（无需API密钥）

```bash
# 使用模拟数据演示系统功能
python demo.py
```

演示模式将：
- 展示Agent的任务规划过程
- 模拟数据采集和分析
- 生成示例报告（保存在 `data/outputs/demo_report.md`）

### 3.4 运行实际采集

```bash
# 调研单个关键词
python main.py -k "美妆"

# 调研多个关键词
python main.py -k "美妆" "护肤" "穿搭"

# 无头模式运行（不显示浏览器窗口）
python main.py -k "美食" --headless

# 限制采集数量
python main.py -k "旅行" --max-notes 50
```

### 4.2 核心模块说明

#### 4.2.1 AI Agent核心 (`agent/core.py`)

实现标准的AI Agent架构：

- **规划器 (Planner)**: 将自然语言需求拆解为可执行的子任务序列
- **执行器 (Executor)**: 执行具体的行动（搜索、浏览、采集、分析等）
- **反思器 (Reflector)**: 评估执行结果，决定下一步行动（继续、重试、跳过、终止）
- **记忆系统 (Memory)**: 提供短期任务记忆，支持去重和上下文理解

#### 4.2.2 浏览器控制器 (`browser/playwright_handler.py`)

基于Playwright模拟真实用户操作：

- 随机延迟（2-5秒，模拟人类思考时间）
- 模拟鼠标移动和滚动行为
- 动态User-Agent切换
- 异常检测和自动恢复

**合规性保证**:
- 仅访问公开可见内容
- 低频访问（2-5秒延迟）
- 不绕过平台技术限制

#### 4.2.3 AI内容分析器 (`ai/content_analyzer.py`)

使用大语言模型进行智能内容理解：

- **内容分类**: 13个预定义类别（美妆护肤、时尚穿搭、美食探店等）
- **广告检测**: AI检测 + 规则增强，准确率 > 90%
- **关键词提取**: 使用GPT-3.5提取TOP 10关键词
- **情感分析**: 判断笔记的情感倾向（正面/中性/负面）
- **热度分级**: 基于点赞、评论、收藏数计算热度分数

#### 4.2.4 数据存储管理器 (`data/data_manager.py`)

负责数据的存储、去重、增量采集：

- **去重机制**: URL去重 + 内容哈希去重（MD5）
- **增量采集**: 加载历史数据，跳过已采集内容
- **多格式输出**: JSON、CSV、Markdown
- **持久化**: 去重缓存自动保存到磁盘

### 7.1 严格遵守的底线

本项目**严格执行**以下合规性要求：

1. ✅ **仅模拟普通用户操作**

   - 随机延迟2-5秒，模拟人类思考时间

   - 不尝试绕过登录墙
   - 不访问需要特殊权限的私有内容
   - 遵守robots.txt规则

3. ✅ **合规、低频次、无暴力爬取**

4. ✅ **不使用任何逆向、抓包、接口破解手段**
