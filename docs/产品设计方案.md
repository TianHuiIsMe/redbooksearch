## 一、项目概述

### 1.1 项目背景

在当前的数字化营销环境中，小红书已成为品牌营销和消费者洞察的重要平台。传统的手工调研方式存在效率低、覆盖面窄、主观性强等问题。本项目旨在构建一套基于AI Agent的自主调研系统，能够模拟真实用户行为，自动化完成小红书平台的 content调研和分析任务。

### 1.2 项目目标

基于模拟正常用户行为（禁止任何逆向、抓包、接口破解手段），实现一套自主运行的AI调研Agent。Agent需接收用户给定关键词，全自动完成整套调研流程：

1. 自主搜索、浏览、采集小红书公开笔记内容
2. 自主清洗、甄别、归类笔记内容
3. 自主统计数据、提炼信息特征
4. 自主生成结构化数据文件 + 分析结论报告

整体全程无人干预、智能决策、自动纠错，体现标准AI Agent「感知-思考-决策-执行」闭环能力。

### 1.3 核心价值

- **效率提升**: 自动化执行调研任务，节省90%以上人工时间
- **覆盖面广**: 可同时调研多个关键词，采集数百条笔记
- **智能分析**: 基于大语言模型的内容理解和特征提炼
- **可追溯性**: 完整记录Agent的每一步思考和决策过程
- **合规性**: 严格遵守平台规则，仅采集公开内容，模拟人类行为


### 2.2 核心组件说明

#### 2.2.1 规划器 (Planner)
- **功能**: 将自然语言调研需求拆解为可执行的子任务序列
- **输入**: 用户需求描述 + 关键词列表
- **输出**: 结构化的任务树（主任务 → 子任务 → 执行步骤）
- **实现**: 基于规则模板 + AI辅助拆解

#### 2.2.2 执行器 (Executor)
- **功能**: 执行具体的行动（搜索、浏览、采集、分析等）
- **核心能力**:
  - 调用浏览器控制器进行页面操作
  - 调用AI分析器进行内容理解
  - 管理执行状态和错误处理
- **实现**: 策略模式 + 命令模式

#### 2.2.3 反思器 (Reflector)
- **功能**: 评估执行结果，决定下一步行动（继续、重试、跳过、终止）
- **评估维度**:
  - 任务完成状态
  - 错误类型和频率
  - 数据采集质量
  - 时间和资源消耗
- **实现**: 规则引擎 + AI辅助决策

#### 2.2.4 记忆系统 (Memory)
- **功能**: 提供短期任务记忆，支持去重和上下文理解
- **存储内容**:
  - 已访问URL集合（去重）
  - 已采集数据列表
  - 错误记录
  - 决策日志
- **实现**: 内存缓存 + 持久化存储

#### 2.2.5 浏览器控制器 (Browser Controller)
- **功能**: 使用Playwright模拟真实用户操作
- **核心特性**:
  - 随机延迟（模拟人类思考时间）
  - 模拟鼠标移动和滚动行为
  - 动态User-Agent切换
  - 异常检测和自动恢复
- **合规性**:
  - 仅访问公开可见内容
  - 低频访问（2-5秒延迟）
  - 不绕过平台技术限制

#### 2.2.6 AI内容分析器 (Content Analyzer)
- **功能**: 使用大语言模型进行智能内容理解
- **分析能力**:
  - 内容分类（13个预定义类别）
  - 广告/种草甄别
  - 关键词提取
  - 情感分析
  - 热度分级
  - 特征提炼
- **支持的模型**: OpenAI API、Azure OpenAI、本地模型（Ollama等）

## 三、功能需求详解

### 3.1 任务自主拆解

**需求描述**: 输入自然语言调研需求，Agent可自行拆解多步骤子任务并有序执行。

**实现方案**:
1. 用户提交需求: "调研小红书平台关于'美妆'和'护肤'的内容"
2. 规划器解析需求，提取关键词: ["美妆", "护肤"]
3. 生成任务树:
   ```
   Main Task: 调研小红书平台关于'美妆'和'护肤'的内容
   ├── Keyword Task 1: 调研关键词: 美妆
   │   ├── Step 1: 分析关键词'美妆'的搜索策略
   │   ├── Step 2: 在小红书搜索'美妆'相关内容
   │   ├── Step 3: 浏览搜索结果列表
   │   └── ...
   └── Keyword Task 2: 调研关键词: 护肤
       └── (类似子任务)
   ```
4. 执行器按序执行任务树

**验收标准**:
- ✅ 支持1-10个关键词的并行拆解
- ✅ 任务拆解准确率 > 95%
- ✅ 支持自定义任务模板

### 3.2 环境感知与自适应执行

**需求描述**: 可识别页面状态、异常卡顿、加载失败、重复数据，自动重试、跳过、终止任务。

**实现方案**:
1. **页面状态识别**:
   - 使用Playwright的`wait_for_selector`等待元素加载
   - 检测页面标题、URL变化判断页面跳转
   - 截图对比判断页面是否加载完成

2. **异常处理机制**:
   - TimeoutError: 增加等待时间后重试（最多3次）
   - ElementNotFound: 滚动页面或切换选择器策略
   - Captcha detection: 暂停任务，提示用户手动处理
   - Network error: 等待5秒后重试

3. **自适应策略**:
   - 动态调整延迟时间（根据页面响应速度）
   - 自动跳过已访问URL
   - 检测到反爬策略时自动降低访问频率

**验收标准**:
- ✅ 自动重试机制覆盖率 > 80% 的异常场景
- ✅ 任务成功率 > 90%（在合规前提下）
- ✅ 异常日志记录完整度 100%

### 3.3 AI智能内容理解与归类

**需求描述**: 依托大模型能力自主完成：内容分类、种草/广告甄别、热度分级、核心信息抽取。

**实现方案**:
1. **内容分类** (13个类别):
   - 使用GPT-3.5/GPT-4进行零样本分类
   - Prompt工程: "请对以下小红书笔记进行分类，可选分类包括..."
   - Fallback: 基于关键词规则的分类器

2. **广告检测**:
   - AI检测: 判断是否存在推广意图、购买链接、合作标识
   - 规则增强: 检测"广告"、"推广"、"种草"等关键词
   - 综合置信度评分

3. **热度分级**:
   - 提取点赞数、评论数、收藏数
   - 计算热度分数: `popularity = likes*1 + comments*2 + collects*1.5`
   - 分级: TOP 10% = 热门, TOP 10-30% = 较热, 其余 = 普通

4. **信息抽取**:
   - 标题、作者、发布时间、正文内容
   - 标签提取
   - 关键实体识别（品牌、产品、地点等）

**验收标准**:
- ✅ 内容分类准确率 > 85%（人工评估100条样本）
- ✅ 广告检测准确率 > 90%
- ✅ 热度分级与真实互动数据相关性 > 0.8
- ✅ 信息抽取完整度 > 95%

### 3.4 记忆与去重能力

**需求描述**: 具备短期任务记忆，自动去重，支持可控增量采集。

**实现方案**:
1. **去重机制**:
   - URL去重: 使用`set`存储已访问URL
   - 内容去重: 使用MD5哈希(标题+作者+内容)检测重复
   - 持久化: 将去重缓存保存到`.dedup_cache.json`

2. **增量采集**:
   - 加载历史数据: 读取`data/raw/raw_YYYYMMDD.json`
   - 对比哈希值，跳过已采集内容
   - 支持按时间范围过滤（如：仅采集最近7天的内容）

3. **任务记忆**:
   - 记录最近N条决策日志（默认50条）
   - 存储错误历史和恢复策略
   - 支持任务恢复（从断点继续）

**验收标准**:
- ✅ 去重准确率 = 100%（无重复数据入库）
- ✅ 增量采集命中率 > 95%
- ✅ 任务中断恢复成功率 > 90%

### 3.5 结果自主输出

**需求描述**: 自动产出：结构化数据表、数据统计维度、可视化结论、调研总结报告。

**实现方案**:
1. **结构化数据** (JSON/CSV):
   ```json
   {
     "note_url": "https://www.xiaohongshu.com/explore/xxxxx",
     "title": "笔记标题",
     "author": "作者名",
     "content": "笔记正文...",
     "tags": ["标签1", "标签2"],
     "likes": 1234,
     "comments": 56,
     "collects": 78,
     "publish_time": "2026-06-20",
     "category": "美妆护肤",
     "is_ad": false,
     "sentiment": "positive"
   }
   ```

2. **数据统计** (Markdown表格):
   - 内容分类分布
   - 广告占比
   - 情感倾向分析
   - 热度TOP 10笔记

3. **可视化结论** (可选):
   - 使用Matplotlib生成饼图（分类分布）
   - 使用WordCloud生成关键词词云
   - 导出为PNG图片

4. **调研报告** (Markdown):
   - 执行概况
   - 主要发现
   - 数据洞察
   - 建议与展望

**验收标准**:
- ✅ 输出格式规范性 100%
- ✅ 数据完整性 > 98%
- ✅ 报告可读性（人工评估）> 85分

### 5.2 核心算法

#### 5.2.1 任务规划算法

```
输入: user_requirement, keywords[]
输出: task_tree

算法步骤:
1. 解析user_requirement，提取调研目标
2. 对每个keyword，生成标准任务模板:
   - 搜索 → 浏览 → 采集 → 分析 → 报告
3. 构建任务依赖图（DAG）
4. 返回可序列化的任务树
```

#### 5.2.2 人类行为模拟算法

```
随机延迟: delay = uniform(min_delay, max_delay)
鼠标移动: 贝塞尔曲线插值，添加随机偏移
页面滚动: 分多次小滚动，随机停留
点击行为: 模拟鼠标移动到目标位置后再点击
```

#### 5.2.3 去重算法

```
输入: note{title, author, content}
输出: is_duplicate (bool)

算法步骤:
1. 生成哈希: hash = MD5(title + "|" + author + "|" + content)
2. 查询哈希缓存: is_duplicate = (hash in content_hashes)
3. 如果不是重复，添加到缓存: content_hashes.add(hash)
4. 定期持久化缓存到磁盘
```
