## 核心架构：大模型 + RPA混合方案

### 1. 架构设计理念

本项目采用**大模型+RPA混合架构**，实现低成本、快速上线、合规安全的医美行业调研解决方案。

```

                          ↓
┌─────────────────────────────────────────────────────────┐
│             大模型规划层（DeepSeek-V3）                     │
```

### 2. 大模型+RPA方案的优势

#### 2.1 无需前端大量开发

**传统方案（爬虫）：**
- 需要分析网页结构
- 需要编写CSS选择器/XPath
- 页面改版后需要重新开发
- 维护成本高

**本方案（大模型+RPA）：**
- 大模型自动理解页面内容
- RPA模拟真实用户操作
- 页面改版后自动适配
- 维护成本极低

**示例：**

```python
# 传统爬虫方案（脆弱）
note_title = page.query_selector('.note-title').inner_text()

# 本方案（智能）
prompt = "请提取当前页面的笔记标题"
note_title = llm.extract_content(prompt)
```

#### 2.2 合规安全，无抓包逆向

**合规保证：**
- ✅ 仅模拟普通用户操作（浏览、滚动、点击）
- ✅ 仅采集公开可见内容
- ✅ 无逆向、无抓包、无接口破解
- ✅ 符合《网络安全法》和平台服务条款

**与传统爬虫对比：**

| 维度 | 传统爬虫 | 本方案（大模型+RPA） |
|------|---------|---------------------|
| 合规性 | ❌ 可能违反平台条款 | ✅ 完全合规 |
| 反爬应对 | ❌ 需要不断对抗 | ✅ 模拟真实用户，不易被检测 |
| 维护成本 | ❌ 高（页面改版需重写） | ✅ 低（大模型自动适配） |
| 开发速度 | ❌ 慢（需要详细分析页面） | ✅ 快（自然语言描述即可） |

```python
def human_like_mouse_move(page, start, end):
    # 生成自然曲线轨迹
    trajectory = generate_bezier_curve(start, end)
```

#### 3.3 滚动行为

```python
# 模拟人类滚动习惯
def human_like_scroll(page):
    # 先快速滚动
    page.evaluate("window.scrollBy(0, 500)")
```
